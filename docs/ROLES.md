# 役割分担 ＆ やることリスト

3 人で並行開発するための担当割り。**自分の担当フォルダの中だけで作業すれば、コンフリクトしません。**
データのやり取りはすべて **Firebase (Firestore)** を介して行います（[ARCHITECTURE.md](ARCHITECTURE.md)）。

| メンバー | 担当領域 | 主な作業フォルダ | ブランチ |
|----------|----------|------------------|----------|
| **はる** | バトルロジック ＆ モーションキャプチャー | `web-parent/`（PC カメラ側） | `feature/haru-battle` |
| **たき**（3回生） | セットアップ ＆ iOS 全般（センサー・振動演出・スマホUI） | `ios-app/` | `feature/ios-setup` |
| **みずき**（1回生） | PC 大画面のフロント UI ＆ 視覚演出エフェクト | `web-parent/`（表示・演出） | `feature/junior-ui` |

## 🟥 はる — バトルロジック ＆ モーションキャプチャー担当

PC の Web カメラで 2 人の動きを認識し、斬撃を判定してスコアを Firebase に送る。

**やること**
- PC の Web カメラを使った **2 人同時の動きの認識** システム構築
- **MediaPipe**（AI ライブラリ）を使った手首の座標追跡・振幅の計算
- 「**一定以上の速度で手が動いたら 1 斬撃**」とする判定ロジックの実装
- 斬撃が成功した際の、**Firebase へのカウント（スコア）送信** 処理
- **iOS ↔ Firestore の連携全般**（Firebase iOS SDK の導入・接続、抜刀ステータスの書き込みなど）も担当。たきが検知したイベントを Firestore に反映する結線はここで行う。

**★ 初手のアクション**
> Mac で Web カメラ（MediaPipe）を起動し、2 人の手の動きをリアルタイムにトラッキングできるかの検証環境を作る。

## 🟦 たき（3回生） — セットアップ ＆ iOS 全般（センサー・振動演出・スマホUI）担当

スマホの構え・抜刀を検知し、武器ごとの振動を返す。`ios-app/` は **UI も含めてたきが一括で担当**（みずきから移管）。リポジトリの土台づくりも担当。

**やること（センサー・振動）**
- **Xcode / Swift（CoreMotion）** を使った、スマホの傾き・加速度の検知
- 「**腰（刀）**」や「**肩の後ろ（大剣）**」に正しくスマホが構えられているかの判定
- シュッと引き抜いた際の「**抜刀完了**」の移動量計算ロジック
- **CoreHaptics** を使った、武器ごとの精密な振動（抜刀時のジャキィン、大剣の重い減衰など）の実装
- 抜刀完了・構え完了などの**イベント／状態を、アプリ内ではるの Firestore 連携に渡せる形で公開**する
  - ※ **Firestore への書き込み（iOS ↔ Firebase の連携）自体は はる が担当**。たきは検知ロジックまで。

**やること（スマホUI ／ ※みずきから移管）**
- **スマホ画面（SwiftUI）**：「**スタート画面**」「**剣を選ぶセレクト画面**」「**抜刀待機画面**」の UI 作成
- 上記 UI と、自分が作るセンサー・振動ロジックの結線（画面遷移と状態の受け渡し）

**★ 初手のアクション**
> Xcode プロジェクト（`ios-app/` フォルダ）を立ち上げ、iPhone 実機を繋いで **CoreHaptics で簡単な振動を鳴らせるか** のテスト。

## 🟩 みずき（1回生） — PC 大画面のフロント UI ＆ 視覚演出エフェクト担当

大画面（PC ブラウザ）の見た目と、バトルの盛り上げ演出に専念する。**スマホUI（SwiftUI）はたきに移管したため担当外**。

**やること**
- **PC 画面（HTML/CSS/JS）**：大画面用の **HP ゲージ・スコアボード** のデザイン、演出画面
- **バトル中の演出**：はるが斬撃を検知した時に、画面にドカンと出る「**斬撃エフェクト**」や「文字演出」の作成

**★ 初手のアクション**
> Web 側（`web-parent/` フォルダ）の土台を作り、PC 画面にテスト用の「**HP ゲージ**」を表示させてみる。

---

## 担当のかぶり（要・声かけ）

- **`ios-app/` … たきが一括で担当**。スマホUIをたきに移管したため、ここの担当かぶりは解消（みずきは触らない）。
- **`web-parent/` … はる（検出・判定）と みずき（表示・演出）が同居**。ファイルを分ければ衝突しないが、触る箇所が近いときは Slack 等でひと声かけてから着手してください。

詳細は [BRANCH_STRATEGY.md](BRANCH_STRATEGY.md)。

---

## 全体の進め方

1. まず最初の数時間で、**各自の「初手のアクション」をクリア**する。
2. その後、**Firebase を介したベースの繋ぎ込みテスト**を行う（最小の値を 1 つ流して、3 者で見えるか確認）。
3. そこからバトルの作り込み・演出強化へ。
