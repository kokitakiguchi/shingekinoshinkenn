## 🟥 はる — バトルロジック ＆ モーションキャプチャー担当

PC の Web カメラで 2 人の動きを認識し、斬撃を判定してスコアを Firebase に送る。

**やること**
- PC の Web カメラを使った **2 人同時の動きの認識** システム構築
- **MediaPipe**（AI ライブラリ）を使った骨格追跡・振幅の計算
- 「**腰（刀）**」や「**肩の後ろ（大剣）**」に正しくスマホが構えられているか判定し `players.pX.ready` / `readyAt` を Firestore に書く（**構え判定は Web 担当**）
- 「**一定以上の速度で手が動いたら 1 斬撃**」とする判定ロジックの実装
- 斬撃が成功した際の、**Firebase へのカウント（スコア）送信** 処理
- タッキーの `drawn` 送信を受け取り、構え完了の `ready` と組み合わせて `drawing → playing` フェーズ遷移を制御

**★ 初手のアクション**
> Mac で Web カメラ（MediaPipe）を起動し、2 人の手の動きをリアルタイムにトラッキングできるかの検証環境を作る。

## 🟦 タッキー（たき）（3回生） — セットアップ ＆ iOS 全般（センサー・振動演出・スマホUI）担当

抜刀を振動付きで実行し、完了を Firestore に送信する。**構え判定は Web 側（はる）が担当**。iOS は抜刀に集中する。

**やること（センサー・振動）✅ 実装済み**
- **CoreMotion**：加速度・傾きの検知、抜刀完了の変位積分（2階積分）ロジック
- **CoreHaptics**：3武器（ライトセーバー・大剣・小剣）ごとの常時ハム・加速度連動・抜刀パターン・振りスパイクを実装
- 抜刀完了時に `shinken_rooms/battle` の `p{n}_ready = true` を **Firestore REST API で直接書く**（Firebase iOS SDK なし）
  - `FirestoreEventSender.swift`・`FirestoreListener.swift`・`FirestoreConfig.swift` 実装済み
  - ※ 構え位置の判定（腰・肩）は**未実装**。現状はボタンを押して「構える」

**やること（スマホUI ✅ 実装済み）**
- `PlayerSelectView.swift`：P1/P2 選択画面（スタート画面）
- `ContentView.swift`：武器選択・加速度ゲージ・構えボタン・抜刀待機・完了バナーを含むメイン画面
- Web の `status="drawing"` を受信したら自動で抜刀待機開始、手動フォールバックボタンも装備

**残タスク（コードフリーズ後の確認用）**
- 🔲 2台同時対戦での `p1_ready && p2_ready` 自動遷移の動作確認
- 🔲 構え位置の判定（腰・肩）— 時間があれば

## 🟩 みずき（1回生） — PC 大画面のフロント UI ＆ 視覚演出エフェクト担当

大画面（PC ブラウザ）の見た目と、バトルの盛り上げ演出に専念する。**スマホUI（SwiftUI）はタッキーに移管したため担当外**。

**やること**
- **PC 画面（HTML/CSS/JS）**：大画面用の **HP ゲージ・スコアボード** のデザイン、演出画面
- **武器選択 UI**：プレイヤーごとの剣選択を行い、Firestore の `players.pX.weapon` に反映する
- **バトル中の演出**：はるが斬撃を検知した時に、画面にドカンと出る「**斬撃エフェクト**」や「文字演出」の作成

**★ 初手のアクション**
> Web 側の土台を作り、PC 画面にテスト用の「**HP ゲージ**」を表示させてみる。

## 担当のかぶり（要・声かけ）

- **`ios-app/` … タッキーが一括で担当**。スマホUIをタッキーに移管したため、ここの担当かぶりは解消（みずきは触らない）。
- **Web 側 … はる（検出・判定）と みずき（表示・演出）が同居**。現行の本体は `web-parent/`（`index.html` / `script.js` / `style.css` / `sounds/`）に集約済み。ルート直下の `a.html` 他は初期プロトタイプ（参考用）。

詳細は [BRANCH_STRATEGY.md](BRANCH_STRATEGY.md)。

## 全体の進め方

1. まず最初の数時間で、**各自の「初手のアクション」をクリア**する。
2. その後、**Firebase を介したベースの繋ぎ込みテスト**を行う（最小の値を 1 つ流して、3 者で見えるか確認）。
3. そこからバトルの作り込み・演出強化へ。
